Movable Type 3.1 - Two Thumbs Down.

I was originally just going to post Keith Robinson’s ’“Movable Type 3.1 Still a Developer Edition?”(Movable Type 3.1 Still a Developer Edition?)’:http://www.dkeithrobinson.com/asterisk/archive/2004/09/mt-31-impressions rant in my Links log, but I want to see if I can get Anil Dash to post in my comments.

I upgraded to 3.1 as soon as it came out, as I am wont to do whenever a new MT release appears. It’s never been a problem before.

This time, however, I enjoyed such luxuries as my carefully crafted (and heftily hacked-upon) templates being overwritten by months-old-version (Thank gawd for Bacula!), and despite upgrading to 3.11, my homepage is still not updating at the right time. (When I make a new post, most pages are re-generated, but not the homepage).

Two thumbs down, Six Apart.

I noticed some funkiness with rebuilds as well. On my site, at least, it appears to be related to the use of background tasks. I see it a lot on individual archives and posting comments. The comment is posted, yet doesn't appear. Strangely enough, after refreshing it does appear. I can only assume that MT3.1 is sending the comment post off into background-land and it's not finished by the time the page is sent again. I did notice there is a directive in the MT3.11 (full) distribution where you can turn off (or on?) background building - perhaps that may help?

On a related note - did you know that if you preview a comment, the preview page does not pick up your fields from the initial comment page, so that if you post at that point, it doesn't appear that it would work (since everything would be blank).
